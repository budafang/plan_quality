# 規劃品質救援任務

這是一個用來教「規劃品質管理」的課堂互動網頁遊戲。

## 教學目標

讓學生透過媒體製作案例理解：

1. 品質不是最高級，而是符合需求。
2. 品質不是最後檢查，而是一開始就要規劃。
3. 品質不是憑感覺，而是用標準、指標與檢核工具管理。
4. AI 時代要加入查證、人工審查、透明性與責任歸屬。

## 使用方式

1. 到 GitHub repo 的 Settings → Pages。
2. Source 選擇 Deploy from a branch。
3. Branch 選 main / root。
4. 儲存後等待 1–2 分鐘，即可取得網址。

## 適合課程

- 專案管理
- 傳播管理
- 媒體製作
- 社群內容企劃
- AI 素養與內容品質管理

## 建議流程

- 8–10 分鐘：學生兩人一組完成遊戲。
- 10 分鐘：討論哪一關最容易選錯。
- 15 分鐘：請學生把自己的作品轉成品質標準、品質指標與檢核表。
